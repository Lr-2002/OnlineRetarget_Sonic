# M1 Deep Read: Tracking, Latent Control, Retrieval, and Contact Metrics

Date: 2026-05-14.

Purpose: close the specific M1 gaps called out in the status audit: BeyondMimic,
MaskedMimic, PHC, ProtoMotions, TMR, and exact contact-metric formulas. This note
records what was actually inspected and how it changes OnlineRetarget's
milestones. It is not a claim that M1 is fully closed; it is enough to make the
next M2Q/M5/M7 choices less ambiguous.

## Direct Design Consequences

1. First online retargeter remains direct G1 motion output, not diffusion. BeyondMimic and MaskedMimic are valuable but both rely on multi-stage control/generative training and are not natural first choices under the 1 ms 4090 inference target.
2. Observation contract should be split into two layers: source-motion retargeter observation and downstream controller observation. Controller-style features from BeyondMimic/ProtoMotions are still relevant sidecars: IMU/base angular velocity, current joints, joint velocities, previous action, and reference/body features.
3. Morphology is not optional. PHC explicitly conditions on body shape when available, and BONES-SEED has per-actor proportional skeletons. M3/M5 should keep actor morphology features in the supervised JSONL path.
4. Review and curation should be behavior-balanced, not only metric-worst. TMR supports semantic clustering/retrieval as a way to avoid reviewing the same near-duplicate behavior many times.
5. Contact metrics now have precise default formulas for M2Q/M4: 3 cm float, 3 cm penetration, 2 cm skate under contact. These should be recorded as defaults/proposals, not universal hard filters, because PHC and PHUMA-style evidence warns that feet/toes and airborne motions need category exceptions.
6. ProtoMotions strengthens the handoff requirement: model export must include observation config/normalization and real-hardware-compatible raw sensor assumptions, not just a checkpoint.

## M2Q / M4 Formula Mapping

Current scanner fields already map cleanly:

| Formula source | Formula | Existing field / gap |
| --- | --- | --- |
| Contact Dynamics floating | contact foot height `> 0.03 m` above ground | `mean_foot_clearance`, `max_foot_clearance`, future `contact_float_rate_3cm` if we add per-frame contact labels |
| Contact Dynamics penetration | foot joint height `< -0.03 m` below ground | `penetration_depth`, current threshold `0.03 m` |
| Contact Dynamics skate | contact foot horizontal displacement `> 0.02 m` while in contact | `contact_skate_rate` and `max_contact_skate_distance` are now metric-only fields in source FK, G1 FK, and M4 offline eval; existing `contact_slide_rate` remains the speed-threshold gate |
| BeyondMimic/PHC body tracking | body position/orientation/linear velocity/angular velocity exponential rewards | M4 has MPJPE/body position scaffold; simulator M7 should add body orientation and velocity errors |
| BeyondMimic/PHC smoothness | previous action/action-rate and energy penalty | M4 can report joint velocity/acceleration/jump; M7 should report action-rate and torque/energy where simulator data exists |

## Still Open

- Need full primary-source extraction for PHUMA thresholds if we want to copy exact numeric filtering values; current repo correctly treats them as proposals only.
- Need decide whether to implement TMR-style semantic clustering with an existing encoder, text labels only, or simple metadata categories first. No new dependency should be added before a concrete review task needs it.
- Need simulator collision/contact labels before self-collision proxy and contact metrics become promotable policy gates.
